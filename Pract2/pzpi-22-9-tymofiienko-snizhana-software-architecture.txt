2.2.	Завдання
Завдання до практичної роботи полягає у вивченні архітектури платформи TikTok — сучасної масштабованої системи для потокової доставки, зберігання та персоналізованої дистрибуції відеоконтенту. Необхідно дослідити, як TikTok забезпечує високу продуктивність, стійкість до навантажень та якість користувацького досвіду завдяки застосуванню таких ключових компонентів: мікросервісна архітектура, розподілені бази даних, системи кешування, мережа доставки контенту (CDN), система рекомендацій на базі машинного навчання, а також хмарна інфраструктура для зберігання й обробки відео.


2.3.	Опис виконаної роботи
1) Що таке Tiktok?
TikTok — мобільна платформа для створення, редагування та перегляду коротких відео, що за останні роки здійснила революцію у способі споживання контенту. Вперше додаток був запущений у Китаї під назвою Douyin, а згодом — для міжнародного ринку під брендом TikTok. Його розробила компанія ByteDance у 2016 році.
Основною метою TikTok є надання користувачам інструментів для швидкого створення відео з музикою, ефектами, фільтрами та можливістю обміну. Однією з ключових особливостей стала стрічка «For You», яка персоналізовано підбирає відео для кожного користувача, базуючись на їх поведінці у додатку.
2) Статистика
Станом на 2024 рік TikTok має понад 1 мільярд активних користувачів щомісяця. Користувачі проводять у середньому 52 хвилини на день у додатку. Платформа стала надзвичайно популярною серед молоді, особливо у віці 16–24 років. За даними Sensor Tower, TikTok посідає 3-тє місце серед найзавантажуваніших додатків у світі у 2023 році, випереджаючи Instagram та Facebook. Використовується в понад 150 країнах світу, підтримуючи десятки мов інтерфейсу.
3) Технології розробки
TikTok використовує низку сучасних мов програмування. Серверна частина написана переважно на Java та Go, що забезпечує високу продуктивність і масштабованість. Python використовується для аналізу даних, обробки відео та елементів машинного навчання. Для фронтенду — React із використанням JavaScript та TypeScript. Мобільні клієнти працюють на Kotlin (Android), Swift (iOS) і C++ (для високопродуктивних модулів відеообробки). Цей стек дозволяє підтримувати кросплатформеність, масштабованість та стабільну продуктивність.
4) Інфраструктура
TikTok побудований на мікросервісній архітектурі, де кожен окремий сервіс відповідає за власну функцію — автентифікацію, зберігання відео, обробку коментарів, стрічку новин, рекомендації тощо.
Ключові компоненти інфраструктури:
• CDN (Content Delivery Network) — дозволяє доставляти відео з мінімальною затримкою, використовуючи найближчі до користувача сервери.
• Хмарна інфраструктура — частково базується на Google Cloud, частково — на приватних дата-центрах ByteDance.
• Балансувальники навантаження — рівномірно розподіляють трафік між сервісами.
• Система CI/CD — дозволяє швидко оновлювати функціонал без збоїв для користувачів.
5) Бази даних
TikTok використовує гібридну систему з кількох баз даних для різних цілей:
• MySQL — основна реляційна база для структурованих даних (акаунти, налаштування, повідомлення).
• Redis — для кешування й швидкого доступу до сесій або популярного контенту.
• Cassandra та HBase — NoSQL-рішення, які дають змогу ефективно масштабувати зберігання відео, метаданих і історії переглядів.
• Hive, Spark, Presto — використовуються для обробки аналітичних запитів, аналізу взаємодій користувачів та трендів.
Це дозволяє TikTok підтримувати як миттєвий доступ до даних, так і масштабну аналітику.
6) Зберігання та обробка даних
Основна складність TikTok — це зберігання та обробка відео та взаємодій у режимі близькому до реального часу. Для цього застосовується:
• Об’єктне сховище (на зразок Amazon S3 або власне рішення ByteDance) — для зберігання відеофайлів, ескізів, обкладинок, аудіо.
• Flink та Apache Kafka — для потокової обробки подій (наприклад, лайків, коментарів, переглядів).
• ML-моделі — щосекундно аналізують тисячі параметрів для формування персоналізованої стрічки.
Підхід TikTok — це "реактивна аналітика": система миттєво реагує на дії користувача, формуючи стрічку динамічно.
7) Монетизація Tiktok
TikTok активно монетизує платформу через кілька каналів. Основний — рекламна платформа TikTok Ads, що дозволяє компаніям таргетовано показувати рекламу. Під час прямих трансляцій користувачі можуть надсилати віртуальні подарунки (Live Gifts), які конвертуються у прибуток для авторів. Також активно розвивається TikTok Shop — платформа для електронної комерції, що інтегрує покупки безпосередньо у відео. Співпраця з брендами та інфлюенс-маркетинг приносять значну частку доходу.
8) Алгоритм рекомендації
Серце TikTok — алгоритм рекомендацій "For You", який формує стрічку з контентом, найбільш релевантним конкретному користувачу.
Алгоритм враховує:
• час перегляду кожного відео;
• кількість взаємодій (лайки, коментарі, репости);
• переглянуті хештеги, аудіодоріжки, жанри контенту;
• інформацію про пристрій і локацію (за згодою).
На базі цього формується вектор поведінки користувача, який оновлюється в реальному часі. Це дозволяє TikTok швидко адаптувати стрічку навіть до нових користувачів за кілька хвилин використання.

2.4.	Висновки
У ході виконання практичної роботи було досліджено архітектуру однієї з наймасштабніших сучасних платформ — TikTok. Розгляд ключових компонентів системи дозволив глибше зрозуміти принципи побудови високонавантажених розподілених сервісів, зокрема: мікросервісну взаємодію, кешування, потокову обробку даних, використання CDN, розподілені бази даних і застосування алгоритмів машинного навчання для генерації персоналізованого контенту.
Особливу увагу було приділено системі рекомендацій, яка є основним фактором залучення користувачів на платформі. Аналіз цієї системи показав важливість адаптивних моделей машинного навчання, що постійно оновлюються відповідно до активності користувачів.
Практична робота також засвідчила переваги мікросервісного підходу до побудови масштабованих застосунків, а також виявила низку викликів, пов’язаних з обробкою великих обсягів відеоконтенту в реальному часі. Здобуті знання та навички є цінними для розуміння сучасних архітектурних рішень у галузі розподілених інформаційних систем і можуть бути використані для розробки власних проєктів з аналогічною структурою.
